# 5. State machine diagram

## 5-1. Userdata State
![1-Userdate-State](https://github.com/seohyun27/breadcast-docs/blob/main/SDS/images/state-machine/1-Userdata-State.jpg?raw=true)
위의 그림은 회원 관련 시스템 상태 변화를 state machine diagram으로 나타낸 것이다.

### 상태 시작
사용자가 서비스 이용을 시작하면 서비스는 회원 미로그인 상태로 진입하여 동작한다.

### 회원 미로그인 상태
1. 사용자가 로그인 상태를 유지할 경우, 시스템은 로그인 상태로 변경된다.
2. 사용자가 회원가입을 요청하면 회원가입 진행 상태로 바뀐다.
3. 회원가입을 완료하면 시스템이 회원을 생성하고, 회원 생성이 완료되면 다시 대기 상태로 돌아간다.
4. 로그인 요청이 들어오면 대기에서 로그인 진행 상태로 이동한다.
5. 로그인이 완료되면 회원 미로그인 상태가 끝나고 시스템이 회원 로그인 상태로 바뀌게 된다.

### 회원 로그인 상태
1. 사용자가 이미 로그인한 채 서비스에 접근했다면 회원 미로그인 상태에서 바로 회원 로그인 상태로 넘어간다.
2. 사용자가 로그아웃을 요청하면 로그아웃 진행 상태가 되고 로그아웃을 완료하면 다시 회원 미로그인 상태로 돌아간다.
3. 회원 탈퇴를 요청하면 회원 탈퇴 진행 상태로 변경된다.
4. 탈퇴 요청이 완료되면 시스템이 회원 삭제를 진행한다.
5. 회원 삭제를 완료하면 회원 미로그인 상태가 된다.

### 상태 종료
사용자가 서비스 이용을 그만두면 회원 미로그인 상태, 회원 로그인 상태에 상관없이 둘 다 상태가 종료된다.


## 5-2. Bakery State
![2-Bakery-State](https://github.com/seohyun27/breadcast-docs/blob/main/SDS/images/state-machine/2-Bakery-State.jpg?raw=true)
위의 그림은 가게 관련 시스템 상태 변화를 state machine diagram으로 나타낸 것이다.

### 상태 시작
사용자가 서비스 이용을 시작하면 서비스는 대기 상태로 이동한다.

### 활성 상태
1. 스크랩한 가게 출력을 요청하면 가게 관심 목록을 출력하는 상태로 전환한다.
2. 가게를 검색하면 관련된 가게 목록을 출력하는 상태로 전환한다.
3. 이렇게 출력한 가게 목록을 통해 시스템을 가게 관련 서비스를 제공하는 상태로 전환할 수 있다.
4. 시스템은 요청에 따라 사용자 작성 리뷰 목록을 출력하는 상태로 전환할 수 있다.
5. 시스템이 리뷰 목록을 출력하는 중 관련 리뷰 관련 입력을 받도록 상태를 전환할 수 있다.

### 가게 관련 서비스 제공 상태
1. 시스템은 사용자의 요청이 있기 전 대기 상태로 대기한다.
2. 가게 정보 출력을 요청하면 시스템은 가게 정보를 출력한다.
3. 시스템은 가게 리뷰 관련 입력을 받도록 상태를 전환할 수 있다.
4. 시스템은 대기 중 요청에 따라 가게 스크랩 여부를 변경하는 상태로 전환한다.
5. 메뉴 목록 출력을 요청하면 시스템은 메뉴 목록을 출력한다.
6. 시스템은 메뉴 목록을 출력하는 중 메뉴 리뷰 관련 입력을 받도록 상태를 전환할 수 있다.
7. 시스템은 제보 관련 입력을 받도록 상태를 전환할 수 있다.
8. 사용자가 가게 관련 서비스를 종료하면 서비스는 이전 상태로 돌아간다.

### 상태 종료
사용자가 서비스 이용을 그만두면 시스템 상태가 종료된다.

6. 시스템은 연관 가게 관련 서비스를 제공하는 상태에서 루트 관련 서비스를 제공하는 상태로 전환할 수 있다.
