# 7. Implementation Requirements

본 시스템은 원활한 서비스 제공 및 유지보수를 위해 다음과 같은 구현 요구사항 및 비기능 요구사항을 정의하고 준수한다.

## 7.1. Implementation Requirements

- 서버(백엔드) 시스템은 Java 21 및 Spring Boot 프레임워크를 기반으로 구현한다.
- 클라이언트(프론트엔드) 시스템은 JavaScript 및 React 라이브러리를 사용하여 개발한다.
- 서버와 클라이언트 간의 데이터 통신은 REST API 방식을 준수하여 구현한다.

## 7.2. Nonfunctional Requirements

### 1) 운영 환경

- 본 시스템은 Windows 운영체제 환경에서의 정상 작동을 기본으로 상정한다.

### 2) 성능

- '검색', '빵지순례', '마이페이지' 등의 주요 메뉴 페이지에 처음 진입하여 데이터가 로딩되는 시간은 5초 이내를 목표로 한다.
- 로그인, 빵지순례 글 작성, 리뷰 등록 등 사용자의 주요 요청(API)에 대한 서버 응답 시간은 3초 이내를 목표로 한다.

### 3) 유저 편의성

- 시스템이 제공하는 모든 주요 기능 메뉴는 화면 좌측의 네비게이션 바(LNB)에 항상 표시되어야 한다.
- 사용자는 좌측 네비게이션 바의 직관적인 메뉴 구조를 통해 원하는 기능에 쉽게 접근할 수 있어야 한다.
- '로그인/로그아웃' 기능과 '기본 검색' 기능은 사용자가 어느 페이지에 있든 쉽게 접근할 수 있도록 모든 페이지의 최상단 공통 헤더(Header)에 항상 존재해야 한다.

### 4) 보안

- 사용자의 비밀번호는 반드시 암호화(Hashing) 처리하여 데이터베이스에 저장한다.
- 인증(로그인)된 사용자만 '마이페이지', '글 작성' 등 관련 API에 접근할 수 있도록 인가(Authorization) 처리를 구현한다.
