# 8. Glossary


| 이름 | 설명 |
| :--- | :--- |
| **서버 (Server)** | Java 21 및 Spring Boot 프레임워크를 기반으로 구현되는 백엔드 시스템입니다. |
| **클라이언트 (Client)** | JavaScript 및 React 라이브러리를 사용하여 개발되는 프론트엔드 시스템입니다. |
| **REST API** | 본 시스템에서 서버와 클라이언트 간의 데이터 통신을 위해 준수하는 통신 방식입니다. |
| **운영 환경** | 시스템이 정상 작동을 기본으로 상정하는 환경으로, Windows 운영체제를 의미합니다. |
| **성능 (Performance)** | 시스템이 준수해야 할 응답 속도 기준으로, 주요 페이지 로딩 시간(5초 이내)과 주요 API 서버 응답 시간(3초 이내)을 목표로 합니다. |
| **유저 편의성** | 사용자가 시스템을 쉽고 직관적으로 사용할 수 있도록 보장하는 요구사항입니다. (예: LNB, 공통 헤더) |
| **네비게이션 바 (LNB)** | 화면 좌측에 항상 표시되어야 하는 메뉴 영역으로, 사용자가 주요 기능에 쉽게 접근할 수 있도록 직관적인 구조를 제공합니다. |
| **공통 헤더 (Header)** | '로그인/로그아웃' 및 '기본 검색' 기능을 포함하며, 모든 페이지 최상단에 항상 표시되어야 하는 공통 영역입니다. |
| **보안 (Security)** | 사용자 비밀번호 암호화(Hashing) 및 API 접근 인가(Authorization) 처리를 포함하는 요구사항입니다. |
| **암호화 (Hashing)** | 사용자의 비밀번호를 데이터베이스에 저장할 때 반드시 적용해야 하는 보안 처리 방식입니다. |
| **인가 (Authorization)** | '마이페이지', '글 작성' 등 인증된 사용자만 접근할 수 있도록 API 접근을 제어하는 처리 방식입니다. |
